# TopicRadar 版本歷程

## 核心原則

1. **專案定位**: 深度報導與專題追蹤工具
   - 目標：為調查記者提供議題進展追蹤與靈感
   - 非即時新聞監控，重視議題脈絡與趨勢分析
   - 更新頻率：每小時一次（國內新聞）、每小時一次（國際新聞）
   
2. **專業導向**: 
   - 介面簡潔專業，不使用顏文字或裝飾性圖示
   - 專注於資訊密度與可讀性
   
3. **版本管理**:
   - 遵循語意化版本號 (Semantic Versioning)
   - 格式：`主版本號.次版本號.修訂號`
   - 每次更新自動更新版本號

### v2.1.0 (2026-01-22)

#### 新增功能
- 多語言關鍵字管理系統（中英日韓四語）

#### 文件更新
- 更新 `CLAUDE.md` 至 v2.1

## 版本號規則

### 主版本號 (Major)
- 重大架構變更
- 不向後相容的改動
- 範例：v1.x → v2.x（引入認證系統）

### 次版本號 (Minor)
- 新增功能
- 向後相容的改進
- 範例：v2.0 → v2.1（多語言關鍵字）

### 修訂號 (Patch)
- 錯誤修復
- 小幅優化
- 文件更新
- 範例：v2.1.0 → v2.1.1
